# File viewer patterns

Patterns are a combination of different primitive components.

Here's a list of components we assembled for the records page:

- File Dropdown: A dropdown with opinionated items to view, download or delete
- File Image: The image / visual component of the file
- File Name: Contains opinionated styling for the names used in the tables
- File Upload: A shared component used for uploading files
- Files Empty: An empty state for cases where no file is uploaded yet
- Files Filter: Logic to filter through files, e.g. by their type
- Files not found: An not-found-state for cases where the user searches for non existing files
- Files Search: An input component to search through uploaded files
